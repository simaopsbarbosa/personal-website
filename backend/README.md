# Backend

This is the Go-based backend, built using the [Gin](https://gin-gonic.com/) web framework and SQLite.

## Project Structure

The project follows standard Go backend conventions, utilizing the `internal/` pattern to ensure a clean separation of concerns and package privacy.

```text
backend/
├── cmd/
│   └── api/
│       └── main.go          # Application entry point
├── internal/
│   ├── database/            # Database initialization and connection logic
│   ├── dto/                 # Data Transfer Objects (Requests/Responses)
│   ├── handlers/            # HTTP handlers and route registration
│   ├── models/              # Database entity models
│   ├── utils/               # Shared utilities (e.g., slug generation)
├── docs/                    # Swagger documentation (generated by swag)
├── seed.sql                 # Database schema and initial seeds
└── go.mod                   # Go module definition
```

### Core Packages

- **`cmd/api`**: Contains the `main.go` file which bootstraps the application, loads environment variables, and starts the server.
- **`internal/models`**: Defines the data structures that map directly to the database tables.
- **`internal/dto`**: Contains Data Transfer Objects.
    - **Requests**: Structs for binding and validating incoming JSON payloads.
    - **Responses**: Structs used to format outgoing JSON, ensuring sensitive fields (like passwords) are never exposed.
- **`internal/handlers`**: Contains the logic for processing HTTP requests. It is organized into functional areas (Auth, Posts).
- **`internal/database`**: Handles the connection to the SQLite database and executes the initial schema.
- **`internal/utils`**: Independent helper functions used across various packages.

## Getting Started

### Prerequisites

- Go 1.25+
- [swag](https://github.com/swaggo/swag) (for documentation updates)

### Configuration

Copy the example environment file and adjust as needed:
```bash
cp .env.example .env
```

### Running the Server

To start the backend server:
```bash
go run cmd/api/main.go
```
By default, the server runs on `http://localhost:8080`.

## API Documentation

The project uses Swagger for API documentation. You can view the interactive UI at:
`http://localhost:8080/swagger/index.html` (when the server is running).

To regenerate the documentation after making changes to handlers or DTOs:
```bash
swag init -g cmd/api/main.go
```

## Database

The project uses SQLite. The schema and initial triggers (for automatic `updated_at` timestamps) are defined in `seed.sql`. The database file (`blog.db` by default) is automatically created on the first run.
